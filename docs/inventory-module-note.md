# Inventory module internal note

Before this work, the demo had no inventory routes, product/stock model, suppliers, purchase orders, or inventory-aware AI. A PPE restock sentence and Shopify/Toast integration cards were the only related references.

The pricing recommendation engine was intentionally not changed. When packaging is revisited in the separate website/workspace projects, Inventory should be recommended for clothing brands, retail, restaurants, salons, esthetics, event centers, construction, plumbing, HVAC, auto repair, e-commerce, and warehousing. Discovery should cover locations, SKU/variant volume, reservations, consumption versus resale, units of measure, expiry/lot needs, barcode/QR needs, and source systems.

This public demo remains deterministic and fictional. It does not persist edits, scan barcodes, connect to POS/e-commerce systems, or sync production data.
